## Java 反序列化 介绍

本系列系统覆盖 Java 反序列化漏洞的基础原理、经典利用链与现代组件漏洞：

### 前置基础

- **Java 反序列化基础**：序列化/反序列化核心概念，`readObject` 反射调用导致的任意代码执行，利用条件与 Fastjson `@type` 对比。
- **字节码安全**：三种字节码加载途径（defineClass / BCEL / URLClassLoader）及 ASM/Javassist 插桩原理，阐明 IAST/RASP 如何利用字节码修改实现无侵入监控。
- **类加载机制**：七阶段生命周期、`Class.forName` vs `ClassLoader.loadClass`、双亲委派源码级解析、TCCL 安全意义及 RASP Bootstrap 注入。

### Commons Collections 利用链

从 CC1 到 CC7 七条经典链的完整构造与递进：CC1 基于 `TransformedMap/LazyMap` + 动态代理；CC2 适配 CC 4.x 以 `PriorityQueue` 为入口；CC3 引入 `TemplatesImpl` 绕过 `InvokerTransformer` 黑名单；CC4 合并 CC2+CC3 双重绕过；CC5 利用 `BadAttributeValueExpException.toString()`；CC6 以 `HashMap` 为入口被称为"通杀链"；CC7 利用 `Hashtable.equals()` 哈希冲突触发。

### 现代组件漏洞

- **Fastjson**：从 1.2.24 到 1.2.68 的 AutoType 绕过演变（L前缀/双写/Class反射/AutoCloseable），以及 Java 反射在漏洞中的核心作用。
- **Log4j (Log4Shell)**：`${jndi:ldap://...}` 表达式直接触发 JNDI 远程类加载，无需方法调用链，从复现到排查修复的完整流程。
- **JNDI**：远程加载对象的五阶段流程（lookup → Reference → 解析 → codebase下载 → 类加载执行），LDAP 与 RMI 恶意服务端实现。
- **PHP 反序列化**：魔法方法自动触发机制、POP 链构造思路、绕过技巧与防御建议。
