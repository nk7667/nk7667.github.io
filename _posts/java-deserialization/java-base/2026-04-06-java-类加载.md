> fastjson-1.2.47的类缓存绕过为何能打破AutoType限制？本质是 `java.lang.Class` 反序列化触发 `Class.forName(...)`，走类加载流程。

## 一、前置基础：Java类加载机制

### 1.1 类的生命周期与加载时机

Java类从加载到内存到卸载出内存，经历七个阶段：**加载 → 验证 → 准备 → 解析 → 初始化 → 使用 → 卸载**。其中验证、准备、解析合称为“连接”。

**主动引用**（会触发类加载）：

- `new`、`getstatic`、`putstatic`、`invokestatic` 指令
- 对类进行反射调用（`Class.forName()` 默认会初始化）
- 初始化子类时父类未初始化

**被动引用**（不触发类加载）：

- 通过子类引用父类静态变量
- `static final` 常量在编译期存入常量池

### 1.2 类加载器层次结构

```
Bootstrap ClassLoader (JVM层，C++实现，加载rt.jar)
        ↑
Extension ClassLoader (加载lib/ext目录)
        ↑
Application ClassLoader (加载ClassPath)
        ↑
Custom ClassLoader (用户自定义)
```

关键点：`Bootstrap ClassLoader`加载的类（如`java.io.File`），`getClassLoader()`返回`null`。

### 1.3 Class.forName() vs ClassLoader.loadClass()

| 方法                                 | 是否执行静态初始化块 | 典型用途                         |
| ---------------------------------- | ---------- | ---------------------------- |
| `Class.forName(className)`         | **是**      | 漏洞利用的“扳机”，执行`static{}`中的恶意代码 |
| `ClassLoader.loadClass(className)` | 否          | 仅加载，不初始化，配合字节码注入场景           |

JVM 中类的完整生命周期包含七个阶段，其中**加载、验证、准备、解析、初始化**是核心五阶段：

```
加载 → 验证 → 准备 → 解析 → 初始化 → 使用 → 卸载
         └──── 连接 ────┘
```

### 2.1 加载（Loading）

JVM 在此阶段完成三件事：

1. 通过类的**全限定名**获取定义此类的二进制字节流（从 class 文件、JAR、网络等）
2. 将字节流的静态存储结构转化为**方法区**的运行时数据结构
3. 在**堆内存**中生成一个 `java.lang.Class` 对象，作为该类的访问入口

```java
// 加载完成后，可以通过 Class 对象访问类的元数据
Class<?> clazz = Class.forName("com.example.User");
```

**关键点**：加载阶段与连接阶段的部分工作（如文件格式验证）会交叉进行，并非严格顺序执行。

### 2.2 验证（Verification）

确保被加载类的正确性，防止恶意 class 文件危害 JVM。分为四个子阶段：

| 验证阶段   | 检查内容                | 失败示例               |
| :----- | :------------------ | :----------------- |
| 文件格式验证 | 魔数、主次版本号、常量池类型      | 篡改 class 文件头       |
| 元数据验证  | 父类继承关系、final 类是否有子类 | 类实现了 `final` 接口    |
| 字节码验证  | 操作数栈、数据类型、跳转指令      | `int` 赋值给 `Object` |
| 符号引用验证 | 被引用的类/字段/方法是否存在     | 调用不存在的方法           |

**安全意义**：某些攻击会尝试提交畸形字节码绕过验证，但现代 JVM 的验证机制非常严格。

### 2.3 准备（Preparation）

为**类变量（static）分配内存并设置**默认零值。

```java
public static int value = 123;
// 准备阶段后，value = 0，而不是 123
// 赋值 123 在初始化阶段执行
```

**特殊情况**：

- `static final` 常量：准备阶段就赋值为代码中的值（编译期常量）
- 实例变量：不在此阶段分配，而是在对象实例化时在堆中分配

### 2.4 解析（Resolution）

将常量池中的**符号引用**替换为**直接引用**。

| 引用类型 | 符号引用示例               | 直接引用示例      |
| :--- | :------------------- | :---------- |
| 类/接口 | `"java/lang/String"` | 方法区中该类的内存地址 |
| 字段   | `"value"`            | 字段在类中的偏移量   |
| 方法   | `"println"`          | 方法入口的指针     |

```java
// 编译期：符号引用 "java/lang/System.out:println"
// 运行时：解析为 JVM 内存中的实际地址
System.out.println("Hello");
```

### 2.5 初始化（Initialization）

执行**类构造器** **`<clinit>()`**，完成静态变量赋值和静态代码块执行。

```java
public class Demo {
    static int a = 10;           // 赋值操作在此阶段执行
    static {                     // 静态代码块也在此阶段执行
        System.out.println("static block");
    }
}
```
