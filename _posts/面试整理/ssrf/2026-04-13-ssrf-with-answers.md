## 正文

### 1.SSRF 原理

服务端提供"代你请求 URL"的能力（下载图片/抓取网页/回调 webhook/解析 OpenGraph 等），若 **请求目标可被用户控制**，攻击者可让服务端访问：

- 内网地址：`http://127.0.0.1:...`、`http://10.0.0.0/8`、`http://192.168.0.0/16` 等
- 云元数据：`http://169.254.169.254/`
- 本地文件/伪协议：`file://`、`gopher://`、`dict://`、`ldap://` 等

### 2.SSRF 修复

- **协议白名单**：只允许 `http/https`，拒绝 `file/gopher/dict/ftp/ldap/sftp` 等危险协议。

- **DNS Pinning（解析后校验并固定 IP）**：

  做 DNS 解析，校验不在内网后进行 **IP 绑定**，后续请求直接使用该校验过的 IP，不再重复解析。并在 **每次重定向** 后再次校验。

  攻击手法：攻击者使一个域名在极短时间内交替返回两个 IP——第一次返回外网安全 IP 通过校验，第二次返回内网攻击目标 IP。若代码分别执行"校验"和"请求"两次独立解析，就会绕过。

  防御手法：**只解析一次**，将解析结果固定（DNS Pinning），后续请求直接使用已校验的 IP。

- **禁止或限制重定向**：避免 30x 跳内网绕过，如必须支持则对每次跳转后的 URL 重新校验。

- **出网隔离**：将发起请求的服务放到受限网络，通过防火墙禁止访问内网与元数据地址。

- **响应控制**：限制响应体大小、Content-Type 白名单、超时/并发，避免成为扫描器/下载器。

### 3.使用协议攻击的具体手段

- **gopher:// → 攻击 Redis**

  `gopher_url="gopher://127.0.0.1:6379/_*3%0d%0a$3%0d%0aSET%0d%0a..."`
  
  `curl -X POST "http://127.0.0.1:8081/api/v1/ssrf/fetch/vuln" --data-urlencode "url=${gopher_url}" -d "weakLevel=1"`

  如果后端客户端支持 gopher，可以构造 Redis 协议的二进制 payload，通过 SSRF 打 Redis 未授权，写计划任务/写 WebShell，从 SSRF 提升到 RCE。

- **dict:// / ldap:// / ftp:// / sftp://**

  这里一般不需要给特别具体的 payload，命令形式讲清楚就够：

  探测内网 FTP：
  `curl -X POST "http://127.0.0.1:8081/api/v1/ssrf/fetch/vuln" -d "url=ftp://192.168.1.10:21/&weakLevel=1"`

  探测内网 LDAP：
  `curl -X POST "http://127.0.0.1:8081/api/v1/ssrf/fetch/vuln" -d "url=ldap://192.168.1.5:389/&weakLevel=1"`

  如果没做协议白名单，这类协议可以被用来探测内网服务，甚至触发更深的 JNDI/LDAP 链。

### 4.SSRF 可以造成哪些危害

1. 作为跳板攻击内网其他主机
2. 无视 CDN，直接访问源站 IP
3. 利用 Redis 未授权访问 getshell
4. 对外网、本地服务器以及内网进行端口扫描，获取服务指纹信息
5. 下载服务器资源，例如用 file 协议读取本地文件
6. 攻击运行在内网或本地的应用程序
7. 对内网 web 应用进行指纹识别，通过访问默认文件方式实现，如 README 文件
8. 攻击内外网的 web 应用，主要是使用 GET 参数就可以实现的攻击，例如 Struts2 漏洞

### 5.SSRF 禁用 127.0.0.1 后如何绕过

**IP 格式变换**

- 将 IP 转为 8/10/16 进制绕过，如 `http://0x7f000001/`
- 将 IP 中的点换成句号或其他字符绕过解析差异
- 特殊 0 绕过：`http://0/`（部分环境如 Windows 下解析为 127.0.0.1，Linux 下行为不定）
- 省略 0：`http://127.1/` 等价于 `127.0.0.1`
- IPv6 格式：`http://[::1]/` 或 IPv4 映射地址 `http://[::ffff:127.0.0.1]/`

**域名 tricks**

- `xip.io` / `xip.name` 类泛解析域名：`http://127.0.0.1.xip.io/`
- `localtest.me` → `127.0.0.1`（长期有效）
- `*.localhost` 域名在某些解析器中会解析到 `127.0.0.1`

**URL 解析差异绕过**

- `http://127.0.0.1@evil.com/` — 不同库取 host 的逻辑不同，有的取 `evil.com`，有的取 `127.0.0.1`
- `http://evil.com#@127.0.0.1/` — `#` 后内容可能被忽略

**其他技巧**

- 添加 `@` 符号绕过：`http://safe.com@127.0.0.1/`
- DNS 重绑定：第一次解析返回外网安全 IP 通过校验，第二次解析返回内网 IP
- 302 跳转绕过：提供一个外网 URL 返回 302 跳转到内网地址，若代码未对跳转后地址重新校验则绕过

### 6.防御方案对比

| 防御手段                        | 能否防 DNS 重绑定 | 是否防 302 绕过 |
| ------------------------------- | ----------------- | --------------- |
| 只允许 http/httpsfo             | 否                | 否              |
| 解析后校验 IP（每次独立解析）   | 否                | 否              |
| DNS Pinning（只解析一次并固定） | 是                | 否              |
| DNS Pinning + 禁用重定向        | 是                | 是              |
| 出站防火墙（网络层）            | 是                | 是              |
